🚀 دستورات اجرا و ابزارهای مورد نیاز
کدهای بالا طوری بهینه‌سازی شده‌اند که سیستم ضد کش گیت‌هاب و پاک‌سازی خودکار پورت‌های مزاحم را در دل خود دارند.

دستور اجرای مستقیم و بدون کش روی هر دو سرور (ایران و خارج)
برای اجرای بدون تداخل از گیت‌هاب، از این دستور هوشمند استفاده کن (به جای USER/REPO آدرس گیت‌هاب خودت را بگذار):

Bash
bash <(curl -H "Cache-Control: no-cache" -H "Pragma: no-cache" -Ls "https://raw.githubusercontent.com/USER/REPO/main/install.sh?v=$(date +%s)")
➕ دستورات اضافی و بسیار حیاتی جهت رفع ابهام کاربران شما:
اگر کاربران شما بعد از زدن اسکریپت باز هم با مشکل عدم پینگ روبرو شدند، این سه دستور طلایی را باید بدانند:

۱. دستور بررسی آزاد بودن پورت در ایران:

Bash
netstat -lntp | grep rathole
(باید تمام پورت‌های انتخابی را در حالت LISTEN نشان دهد).

۲. دستور آزاد کردن کامل فایروال لینوکس (اگر فایروال کلود و دیتاسنتر مسدود نباشد):

Bash
sudo ufw disable
(اگر اوبونتو ۲۴ همچنان ترافیک کلاینت‌ها را بلاک کند، با زدن این دستور فایروال داخلی موقتاً خاموش می‌شود تا تست سلامت انجام شود).

۳. یادآوری اجباری 3X-UI در خارج:

⚠️ نکته کلیدی: کاربران حتماً باید در پنل X-UI خارج، وارد بخش Inbounds شده، کانفیگ را ادیت کنند و گزینه Listen IP را کاملاً خالی بگذارند یا روی 0.0.0.0 تنظیم کرده و دکمه قرمز رنگ Restart Panel را بزنند؛ در غیر این صورت پینگ کانفیگ کور خواهد ماند.
